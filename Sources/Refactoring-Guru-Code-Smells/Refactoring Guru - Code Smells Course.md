# Refactoring Guru - Code Smells Course

**Course Description:**
Comprehensive guide to code smells based on Refactoring Guru resources. Covers different types of code smells, how to identify them, and refactoring techniques to eliminate them. Helps developers write cleaner, more maintainable code.

## Course Structure

### Overview

- [[Programming/Code smells/Types of code smells]] - Introduction to code smells and their categories

### Code Bloaters

Signs that code has grown too large:

- [[Programming/Code smells/code bloaters]] - Overview of bloater code smells
- [[large class]] - Classes that have grown too large
- [[long method]] - Methods that do too much
- [[long parameter list]] - Methods with too many parameters
- [[Programming/Code smells/data clumps]] - Groups of data that should be objects
- [[primitives obsession]] - Overuse of primitive types instead of objects

## Learning Path

1. Understand what code smells are and why they matter
2. Learn to identify different types of code smells
3. Recognize code smells in your own code
4. Apply refactoring techniques to eliminate smells
5. Prevent code smells from appearing in new code

## Key Concepts Covered

- **Code Smells**: Indicators of deeper problems in code
- **Bloaters**: Code that has grown too large
- **Object-Orientation Abusers**: Misuse of OOP features
- **Change Preventers**: Code that makes changes difficult
- **Dispensables**: Unnecessary code
- **Couplers**: Excessive coupling between classes

## Refactoring Techniques

- Extract Method
- Extract Class
- Replace Parameter with Object
- Replace Primitive with Object
- Move Method/Field
